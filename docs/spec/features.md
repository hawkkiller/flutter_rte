# Features

This section describes the features that the project should have.
This is a work in progress and is a subject to change.

## Text Formatting

The RTE should support various text formatting options, such as bold, italic, underline, and strikethrough.
It should be extensible, meaning that it should be possible to add more formatting options as the project evolves.

## Custom Block Widgets

The RTE should support custom block widgets, such as images, videos, and other custom widgets.
It should be possible to add custom block widgets to the RTE and customize their appearance and behavior.

## Inline Widgets

The RTE should support inline widgets, such as links, mentions, hashtags, and other custom inline widgets.

## Extensible API

The RTE should have an extensible API that allows developers to customize its behavior and appearance.
It should be possible to change the default behavior of the RTE, add custom formatting options, and customize the
appearance of the RTE.

## Selection and Cursor

The RTE should support selection and cursor manipulation, such as selecting text, moving the cursor, and selecting
block widgets.
It should support selection of multiple blocks, as well as multiple inline widgets.

### Popover

It should be possible to show a popover when a user selects text or block widgets.

## Command System

Every action in the RTE should be represented as a command. This allows developers to create custom commands and
customize the behavior of the RTE. For example, developers should be able to create custom commands for formatting
text, inserting block widgets, and manipulating the selection.

### Undo/Redo

The RTE should support undo and redo functionality, allowing users to undo and redo their actions.

## Collaboration

The RTE should support collaboration features, such as real-time editing, presence indicators, and conflict resolution.
